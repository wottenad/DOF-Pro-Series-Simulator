# DOF Pro-Series Simulator

**A browser-based digital twin for virtual pinball cabinet lighting and effects.**

## What Is This?

[![DOF Pro-Series Simulator Demo](https://img.youtube.com/vi/_gf0vMBl8-Q/maxresdefault.jpg)](https://www.youtube.com/watch?v=_gf0vMBl8-Q)

The DOF Pro-Series Simulator is a real-time, in-browser visualization tool for the [DirectOutput Framework (DOF)](https://github.com/DirectOutput/DirectOutput) — the open-source LED and toy control system used in virtual pinball cabinets worldwide.

It parses your actual DOF configuration files and renders your LED matrix effects, addressable LED strips, RGB toys, and physical toy indicators in real time — **no hardware required**. Design, preview, and refine your cabinet's entire lighting setup from any browser.

### Key Features

- **Full Cabinet Simulation** — Renders your LED matrix (any resolution), all addressable LED strips, RGB toys, and physical toy indicators using the same configuration files your actual hardware uses.
- **DOF Builder** — A multi-layer visual effect designer. Pick colors from the full 137-color DOF palette, set timing parameters, configure area coverage, add motion scrolling, sparkle effects, and shape sprites — all with real-time preview. Generates ready-to-paste DOF code strings.
- **Animation Simulator** — For Pinup Popper users: loads your menu effects spreadsheet and GIF animation file to preview how every E-code entry will look on your matrix and strips during menu navigation.
- **Code Simulator** — Paste any DOF effect string and fire it with momentary, latched, or timed repeat triggers. Effects render over whatever is already playing — ROM table effects or Anim Sim output.
- **Layer & Toy Inspectors** — Surgical analysis tools. The Layer Inspector shows all effects for a single trigger organized by Z-layer. The Toy Inspector shows all triggers that affect a single toy.
- **9 Color Themes** — Blueprint (default), Dark, Light, Retro, Matrix, Solarized Dark, One Dark, Monokai, and Catppuccin Mocha.

## Quick Start

### Prerequisites

- A modern web browser (Chrome, Firefox, Edge, or Safari)
- Your DOF configuration files (see [Required Files](#required-files) below)
- Sample files are available in the Help & Reference section

### Running the Simulator

1. Download or clone this repository.
2. Open `index.html` in your browser. **No server required** — it runs entirely client-side.
3. Load your configuration files using the Configuration panel.
4. Click **✓ Confirm Files & Continue** to unlock the simulator.

### Required Files

All files are generated by the [DOF Config Tool](http://configtool.vpuniverse.com/) and stored in your local DOF installation (typically `C:\DirectOutput\Config\`).

| File | Source | Required? |
|------|--------|-----------|
| `Cabinet.xml` | `C:\DirectOutput\Config\` | **Yes** |
| `DirectOutputConfig.ini` | `C:\DirectOutput\Config\` | **Yes** |
| `DirectOutputConfig30.ini` | `C:\DirectOutput\Config\` | **Yes** |
| `DirectOutputConfig2.ini` | `C:\DirectOutput\Config\` | Optional — RGB toys |
| `Cabinet.json` | Export from DOF Config Tool | Optional — combo definitions |
| `DirectOutputShapes.xml` | DOF Config Tool download | Optional — shape sprites |
| `DirectOutputShapes.png` | DOF Config Tool download | Optional — shape atlas |

**Pinup Popper users** who want to use the Animation Simulator also need:
- `PUPDatabase.db` — from your Pinup Popper installation root
- Pinup Menu Effects spreadsheet (`.xlsx` or `.csv`) (Download sample files in Help & Reference section)
- Pinup Menu GIF animation file (`.gif`) (Download sample files in Help & Reference section)

> **Tip:** Copy all files into a single folder (e.g., `DOF Mod Files`) for easier loading.

## Application Sections

### Simulator View

The default three-panel workspace:

| Panel | Contents |
|-------|----------|
| **Left** | Configuration panel (file loading), Table View (ROM/trigger selection), Animation Simulator |
| **Center** | LED matrix display, LED strip rack |
| **Right** | Active toys & indicators, RGB toys, physical toy checkboxes |

### DOF Builder

Switch to the Builder by clicking the **◆ DOF Builder** button in the header. The Builder provides:

- **6 compositing layers** (L1–L6) with independent Z-order
- **137-color DOF palette** with click-to-select
- **Timing controls** — 5 effect types (Static, Blink, Fade In, Fade Out, Fade In/Out)
- **Area & Motion** — position, size, scroll speed, and direction
- **Sparkle** — density, min/max on-time for random pixel twinkle
- **Shapes** — sprite-based patterns from the DirectOutputShapes library
- **13 target sections** — 8 MX (full effects) + 5 RGB-only (color only)
- **Staged Output Table** — tracks all configured sections with generated DOF strings

Builder state persists in localStorage between sessions.

### Code Simulator

Paste any DOF effect string into the Code Sim panel and test it with three trigger modes:

- **Momentary** — fires while held
- **Latched** — toggle on/off
- **Repeat Timer** — fires on a configurable interval

Effects render layered over existing ROM table effects or Anim Sim output.

## Technology

- **Vanilla JavaScript** — no frameworks, no dependencies, no build step
- **CSS Grid + Flexbox** — responsive three-panel layout
- **Canvas API** — shape sprite rendering from the DirectOutputShapes atlas
- **requestAnimationFrame** — 60fps rendering loop for all matrix, strip, and toy animations
- **FileReader API** — client-side XML, INI, JSON, XLSX, CSV, GIF, SQLite parsing
- **localStorage** — Builder state persistence and theme preference

## Workflow: From Builder to Cabinet

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  DESIGN  │───▶│   TEST   │───▶│  DEPLOY  │───▶│   PLAY   │
│ (Builder)│    │(Code Sim)│    │(Config   │    │(Cabinet) │
│          │    │          │    │  Tool)   │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

1. **Design** — Open the DOF Builder and configure your multi-layer effect.
2. **Test** — Send to Code Sim. Verify against existing table effects or Anim Sim output.
3. **Deploy** — Copy the DOF string to the [online DOF Config Tool](http://configtool.vpuniverse.com/). Assign to the correct E code.
4. **Play** — Download the updated INI files and place them in your DirectOutput config folder.
